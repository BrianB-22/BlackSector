# Port System Specification

## Version: 0.2

## Status: Draft

## Owner: Economy Systems

## Last Updated: 2026-03-03

---

# 1. Purpose

The Port System defines the primary economic interaction points within the universe.

Ports serve as locations where ships may:

* buy commodities
* sell commodities
* refuel
* repair
* interact with economic infrastructure

Ports generate economic activity that drives:

* trade routes
* AI trader movement
* player profit strategies
* regional economic imbalance
* conflict hotspots

The Port System provides the core **commodity supply and demand engine** for the universe economy.

---

# 2. Scope

IN SCOPE:

* Commodity trading
* Port commodity inventories
* Supply and demand models
* Commodity pricing
* Port production and consumption
* Port classes
* Trade transactions
* Commodity definitions loaded from configuration

OUT OF SCOPE:

* Player inventory management (Player System)
* Ship cargo capacity (Ship System)
* AI trading behavior (AI Trader System)
* Sector ownership and faction control (Faction System)
* Resource extraction (Mining System)

---

# 3. Design Principles

Non-negotiable constraints:

* Server authoritative pricing
* Deterministic inventory changes
* Tick-driven production
* Finite commodity inventories
* Commodity definitions stored as external configuration
* Regional economic imbalance encouraged
* Trade routes should naturally emerge

Ports must never function as infinite commodity sources.

---

# 4. Core Concepts

Port
A static structure within a sector that participates in the economic simulation.

Commodity
A tradable resource used for economic exchange.

Commodity Definition
A configuration-driven description of a tradable resource.

Commodity definitions include:

* name
* base price
* weight
* category
* rarity
* economic tags

Commodity definitions are loaded from configuration files and remain immutable during runtime.

Inventory
The quantity of a commodity available at a port.

Supply
Commodity surplus available for purchase.

Demand
Commodity shortage that a port will purchase.

Production
Commodity generated by a port during each economic tick.

Consumption
Commodity removed from a port during each economic tick.

# 5. Data Model

## Entity: Port (Persistent)

* port_id: uint64
* sector_id: uint64
* port_class: enum
* infrastructure_level: int

---

## Entity: CommodityDefinition (Configuration)

Commodity definitions are loaded from JSON configuration at server startup.

Fields:

* commodity_id: string
* name: string
* category: enum
* base_price: float
* weight: float
* rarity: float
* tags: list<string>

Commodity definitions are immutable during runtime.

---

## Entity: PortCommodityState (Persistent)

Represents commodity inventory within a specific port.

Fields:

* port_id: uint64
* commodity_id: string
* inventory_current: int
* inventory_capacity: int
* production_rate: int
* consumption_rate: int

---

## Entity: CommodityTransaction (Transient)

Represents a completed trade action.

Fields:

* ship_id: uint64
* port_id: uint64
* commodity_id: string
* quantity: int
* price_per_unit: float
* transaction_tick: int

# 6. State Machine

Ports typically remain operational but may enter alternative states.

ACTIVE
→ Port operates normally.

DEPLETED
→ Commodity inventory exhausted.

OVERSTOCKED
→ Inventory exceeds optimal levels.

DISABLED
→ Port temporarily inactive due to damage or scripted events.

State transitions may be triggered by:

* economic depletion
* sector conflict
* infrastructure damage
* anomaly effects

---

# 7. Core Mechanics

Commodity inventories update during the economic phase of each server tick.

Economic tick sequence:

1. production added to inventory
2. consumption removed from inventory
3. price recalculation
4. AI trade opportunity evaluation

Player trade interaction flow:

1. ship docks at port
2. server retrieves port inventory state
3. player selects buy or sell action
4. cargo inventory validated
5. commodity quantity transferred
6. port inventory updated
7. transaction recorded

Ports generally:

* produce some commodities
* consume others
* rarely both for the same commodity

# 8. Mathematical Model

## Variables

InventoryLevel
Current quantity of a commodity at a port.

InventoryCapacity
Maximum quantity the port can store.

SupplyRatio
InventoryLevel ÷ InventoryCapacity

BasePrice
Standard commodity value defined in CommodityDefinition.

---

## Pricing Formula

PricePerUnit =

BasePrice × DemandMultiplier × ScarcityMultiplier

Where:

DemandMultiplier =
1 + (1 - SupplyRatio)

ScarcityMultiplier =
1 + (0.5 × (1 - SupplyRatio))

Price clamps:

MinimumPrice = BasePrice × 0.5
MaximumPrice = BasePrice × 3.0

# 11. Failure & Edge Cases

Insufficient Inventory
Buy order rejected.

Cargo Capacity Exceeded
Transaction rejected.

Invalid Commodity ID
Request rejected.

Simultaneous Transactions
Resolved using server-side transaction locking.

Port Disabled
Trading unavailable until state resolves.

---

# 12. Performance Constraints

Economic updates must support:

* up to 1,000 ports
* 50–100 concurrent players transacting

Inventory updates must occur during the economic tick phase.

Price calculations must remain O(1).

---

# 13. Security Considerations

All commodity prices must be calculated server-side.

Clients cannot:

* modify price
* modify inventory
* bypass cargo limits
* alter commodity definitions

Commodity configuration files must only load during server startup.

---

# 14. Telemetry & Logging

Track metrics including:

* trade volume per commodity
* most profitable trade routes
* port depletion frequency
* commodity scarcity events

Log events:

* large trade transactions
* inventory depletion
* extreme price fluctuations

---

# 15. Balancing Guidelines

Economic behavior should encourage:

* long-distance trade routes
* regional scarcity
* dynamic market opportunities
* exploration-driven trade

Design targets:

* profitable routes require travel
* high-risk regions provide higher-value markets
* safe regions provide stable but low-margin trade

---

* player-owned ports
* complex manufacturing chains
* commodity derivatives
* player trade contracts
